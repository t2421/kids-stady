# 出題カリキュラム仕様 (小1〜小6)

学習指導要領 (算数) の「数と計算」領域を軸に、3択タップで解答できる形へ落とした出題仕様。
**この文書が正典**。生成器 (`src/lib/curriculum/gradeN.ts`) とテストはこれに従う。

## 共通ルール

- `Problem` 型 (`types.ts`) に従う。answer/choices は**文字列** (分数 "3/5"、小数 "0.7"、あまり "4あまり2"、単位付き "70%" など)
- choices は常に3つ・ユニーク・answer を含む。ディストラクタは「ありがちな間違い」を優先
  (くり上がり忘れ ±10、九九の隣 (n±1)×m、通分忘れ、位取りミス ×10/÷10 など)
- 表記: ひらがな中心。学年進行に応じて cm/mm/kg/km/L/%/x などの記号はそのまま使う
- くりあがり/くりさがり (小1) のみ cherry ヒントあり。他は hint: null
- explain は 1〜3行の短いステップ (子供が読める言葉で)

## テスト要件 (プロパティテスト、各スキル300問以上)

1. answer ∈ choices、choices 3つユニーク、text/explain 非空
2. 各スキルの値域・制約 (上表) を検証
   - 例: g2_add2 → 和<100 / g3_div_amari → あまり1〜b-1 / g5_bunsu_add → 既約
   - 分数系は「既約であること」を gcd で検証
3. 分数・あまり等の文字列フォーマット検証 ("aあまりb", "a/b", "xじyぷん")
